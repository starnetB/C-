一，类图

类图中，每个类由3个部分组成，上面的是类名，中间的是属性，下面的是方法。

属性和方法可以用修饰符修饰：加号（+）表示public；减号（-）表示private；井号(#)表示protected

二，类的6种关系

类的6种关系：泛化 实现 组合 聚合 关联 依赖

6种关系依次减弱。

参考：UML类图几种关系的总结

1，泛化

泛化是一种继承关系，如老虎类继承动物类，动物是老虎的泛化，老虎是动物的特化。

  1. class Animal {
  2. public:
  3.     Animal() {
  4.         posX = posY = 0;
  5.     }
  6.     void move(int dx, int dy) {
  7.         posX += dx, posY += dy;
  8.     }
  9.     void getPos() {
  10.         cout << "x=" << posX << ", y=" << posY << endl;
  11.     }
  12. private:
  13.     int posX, posY;
  14. };
  15.  
  16. class Tiger :public Animal {
  17. public:
  18.     Tiger() {
  19.         this->name = "Tiger";
  20.     }
  21.     string name;
  22. };
  23.  
  24. int main()
  25. {
  26.     Tiger t;
  27.     cout << t.name << endl;
  28.     t.move(3, 4);
  29.     t.getPos();
  30.     return 0;
  31. }

2，实现

实现关系是子类实现了接口类。

在JAVA中，有专门的接口类和实现的关键字。

在C++中，用纯虚类作为接口类，而且纯虚类也没有关键字，包含纯虚函数的就是纯虚类。

 老虎类实现了攀爬接口类。

  1. class Animal {
  2. public:
  3.     Animal() {
  4.         posX = posY = 0;
  5.     }
  6.     void move(int dx, int dy) {
  7.         posX += dx, posY += dy;
  8.     }
  9.     void getPos() {
  10.         cout << "x=" << posX << ", y=" << posY << endl;
  11.     }
  12. private:
  13.     int posX, posY;
  14. };
  15.  
  16. class Climb {
  17. public:
  18.     Climb() {
  19.         posZ = 0;
  20.     }
  21.     virtual void GoClimb(int dz) = 0;
  22.     void getHeight() {
  23.         cout << "z=" << posZ << endl;
  24.     }
  25. protected:
  26.     int posZ;
  27. };
  28.  
  29. class Tiger :public Animal, public Climb {
  30. public:
  31.     Tiger() {
  32.         this->name = "Tiger";
  33.     }
  34.     string name;
  35.     void GoClimb(int dz) {
  36.         posZ += dz;
  37.     }
  38.  
  39. };
  40.  
  41. int main()
  42. {
  43.     Tiger t;
  44.     cout << t.name << endl;
  45.     t.move(3, 4);
  46.     t.getPos();
  47.     t.GoClimb(5);
  48.     t.getHeight();
  49.     return 0;
  50. }

3，组合

组合是整体与部分的关系，个体没有独立的生命周期。

组合是把个体对象作为整体类的属性。

 

组合关系中，有1对1的，也有1对多的。 

每只老虎有4条腿，1个头。

  1. class Leg {
  2. protected:
  3.     int len;
  4. };
  5. class Head {
  6. protected:
  7.     int size;
  8. };
  9.  
  10. class Tiger {
  11. public:
  12.     Tiger() {
  13.         this->name = "Tiger";
  14.     }
  15. private:
  16.     string name;
  17.     Leg leg[4];
  18.     Head head;
  19. };

4，聚合

聚合也是整体与部分的关系，但个体有独立的生命周期。

聚合是把个体对象的指针作为整体类的属性。

 

 虎群里可能有1-n只老虎。

  1. class TigerGroup {
  2. private:
  3.     vector<Tiger*> vtiger;
  4. };

5，关联

关联是一种拥有关系，它使一个类知道另一个类的属性和方法，

如老虎拥有若干食物，那么它就知道每个食物是什么，什么颜色，有没有坏掉等等。

有1-n的单向关联，也有n-n的双向关联，双向关联可以用双向箭头，也可以没有箭头。

  

  1. class Food {
  2. protected:
  3. 	string foodName;
  4. 	string foodColor;
  5. };
  6.  
  7. class Tiger {
  8. public:
  9. 	Tiger() {
  10. 		this->name = "Tiger";
  11. 	}
  12. private:
  13. 	string name;
  14. 	Leg leg[4];
  15. 	Head head;
  16. 	vector<Food> food;
  17. };

在《大话设计模式》中企鹅和气候的关系也是关联，理由是企鹅需要知道气候的变化。

依赖是一种使用关系，如老虎喝水。

  28. 	}
  29. private:
  30. 	string name;

  34. };
